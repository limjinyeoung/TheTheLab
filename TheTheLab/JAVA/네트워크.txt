포트 (프로세스, 서비스의 주소): 서버 안에서 서비스를 제공하는 프로세스의 주소 (16BIT)
	서비스의 종류에 따라서 약속된 포트 
		-80 : web server
		-22 : SSH
		-21 : FTP

사용자들은 서비스를 제공하는 컴퓨터의 IP주소(컴퓨터의 주소)를 기억하지 못하기 때문에
125.209.222.141  =>  naver.com 로 변경한다.

DNS (domain name server) : naver.com을 IP주소로 변경한다.








프로세스
	-서비스 (데몬) : 백그라운드 프로세스


유닉스(리눅스) :작은 것이 아름답다. / 모든 것은 파일이다.(모든것은 파일의 인터페이스를 통해 다룰 수 있다.
					=>A프로세스가 B프로세스에 데이터를 전송할 때 write/OutputStream, 수신할 때 read/InputStream


socket : 네트워크 파일 // 소켓을 통해서 데이터를 읽으면, 네트워크 데이터를 수신한다. / 소켓에 데이터를 쓰면, 네트워크 데이터를 전송한다.
	socket은 TCP/IP뿐 아니라 다양한 프로토콜에서 데이터를 교환하기 위해 사용되는 특수한 파일이다.

server // 클라이언트의 접속을 수락한다.(server socket) -> 세션이 형성된다. -> 세션을 통해 데이터를 주고받을 수 있다.(socket)


localhost : 자기 자신의 호스트네임(도메인 네임)                                127.0.0.1 (자신의 IP주소)

SRP (Single Responsibility Principle) 
단일 책임의 원칙 : 하나의 클래스는 하나의 역할만 충실하게 해야한다.

OCP (Open Close Principle)
개방-폐쇄의 원칙 : 하나의 클래스는 수정에는 폐쇄되어있고, 확장에는 개방되어 있어야 한다.

LSP (Liskov Substitution Principle)
리스코프 치환 원칙 : 동일한 클래스를 상속하는 경우, 자식클래스는 부모클래스로 대체 가능해야 한다.

ISP (Interface Segregation Principle)
인터페이스 분리의 원칙 : 사용자에게 필요한 인터페이스를 분리시킨다.

DIP (Dependency Inversion Principle)
의존관계 역전 원칙 : 다른 클래스보다 인터페이스에 의존하는 코드가 유연하다.
